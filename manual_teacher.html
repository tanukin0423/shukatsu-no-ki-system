📋システム全体像
🌳 就活の木 面談予約システムとは

学生がQRコードから面談を予約し、先生がメールで受け取って日程を確定、カルテを作成するまでを一貫してサポートするシステムです。 すべてのデータはFirebase（クラウド）に保存されるため、大学・自宅・どこからでも最新状態でアクセスできます。
さらに、大学のExcelを事前に取り込むことで、予約・merge前でも学生の基本情報をカルテに即表示できます。

📊 student_master_import.html 事前作業
大学ExcelをFirebaseに一括登録。3年生・4年生対応。これを事前に実施することで全ツールの汎用性が大幅に向上。
🗓 admin.html
管理者画面。カレンダーの空き枠管理（×設定）。相談員の追加・編集。QRコード生成。
📋 NO2NO3_merged_2026.html
面談管理ツール。メール解析→日程確定→メール一括送信。確定データを自動でFirebaseに保存。
📝 karte_v5.html
面談カルテ。学籍番号を入力するだけで、面談カルテ OR 学生マスターの基本情報を即表示。merge前でも使用可能。
📅 学生向け予約フォーム
📅 index.html（学生用・QRコードで配布）
学生が担当先生・日程・相談内容を入力してメール送信。Firebaseのカレンダー設定が自動反映され、×の日時は選べない。

①student_master_import.html の使い方
🎯 このツールを事前に使うと何が変わるか
大学Excelの全学生情報をFirebaseに登録しておくことで、予約メールが届く前・merge前でも学籍番号を入力するだけでカルテの土台（氏名・ゼミ・メール・担当職員・就職ステータス等）が自動表示されます。
📊 操作手順（3ステップ）
🔑
ログイン（パスワード：karigu2026）
管理者パスワードでログインします。GitHub Pages版（https://）でのみFirebase登録が動作します。
📂
Excelファイルを読み込む
3年生（28卒）と4年生（27卒）のExcelファイルをそれぞれ選択します。
どちらか一方だけでもOKです。
「📊 データを解析する」ボタンを押すと、14項目が自動抽出されてプレビュー表示されます。

抽出される項目：学籍番号・氏名・フリガナ・学年・性別・学科・ゼミ・担当職員・就職希望・ステータス・携帯電話・メールアドレス・在留資格・国籍
☁️
「☁️ 全件をCloudに登録する」を押す
Firebaseの「students」コレクションに学籍番号をキーとして一括登録されます。
プログレスバーで進捗が確認できます。同じ学籍番号は上書きされます。

フィルター登録も可能：氏名・学籍番号・ゼミで絞り込んだ後「☁️ 表示中のデータのみ登録」でフィルター結果だけ登録できます。
💡 Cloud検索・確認機能
画面下部の「Cloudデータ検索・確認」で、学籍番号を入力して登録済みデータを確認できます。誤って登録したデータは「🗑 削除」ボタンで個別削除できます。
②admin.html の使い方
🗓 カレンダー設定（空き枠の管理）
🔑
ログイン（kitayama / kitayama2026）
ログイン時にFirebaseから最新データを自動読み込みします。「🔥 Firebase接続済み」と表示されれば正常です。
❌
塞がっている時間帯を「×（赤）」にする
サイボウズを並べて確認しながら操作します。
○（緑）＝予約可能　×（赤）＝予約不可
時間帯は45分刻み：10:00 / 10:45 / 11:30 / 12:15 / 13:00 / 13:45 / 14:30 / 15:15 / 16:00
「この週を全て×」ボタンで丸ごとブロックも可能。60日以降は設定不可（黄色表示）。
💾
「💾 カレンダーを保存」を押す
Firebaseに即時保存。「✅ Firebase + ローカルに保存しました」と表示されればOKです。index.htmlに即座に反映されます。
📱 QRコードの生成と配布
📱
「カレンダー設定」タブ下部 → GitHub PagesのベースURLを入力
例：https://tanukin0423.github.io/shukatsu-no-ki-system/
入力するとQRコードが自動生成されます。
⬇️
「QRコードをダウンロード」または「印刷」して学生に配布
QRを読み取ると担当が自動で「北山先生」に設定されます。
③NO2NO3_merged_2026.html の使い方
📨 STEP1：メールを貼り付けて解析する
🔑
ログイン（kitayama / kitayama2026）
ログイン後、自分のカレンダーが自動で読み込まれます。
📋
「受信解析」タブ → メール本文をペースト → 「解析して追加」
複数名分を繰り返し貼り付けてOKです。全員分貼り付けてから次に進んでください。初回・2回目以降どちらにも自動対応します。
📅 STEP2：振り分けと日程確定
🚀
「日程確定」タブ → 優先ルール選択 → 「振り分け開始」
緑 第1希望で確定　青 第2・第3希望で確定　赤 全希望が埋まっている（要再調整）

☁️ この時点で全学生のカルテ基本情報がFirebaseに自動保存されます！
画面上部に「☁️ Cloud保存完了！○名のカルテ基本情報を保存しました」のバナーが表示されます。
✏️
プルダウンで手動調整 → 「🔄 調整後・再判定」
変更後は再判定で重複チェック。再判定後もFirebaseが自動更新されます。
📧 STEP3：確定メールを送信する
✅
「全員のメールを一括作成する」→ 1件ずつ確認して送信
緑 → 面談確定メール
青 → 調整完了メール（または再調整依頼）
赤 → 再調整依頼メール
💡 CSVについて
「CSV出力」タブからCSVのダウンロードもできます（バックアップ用）。「今すぐFirebaseに再保存」ボタンで手動再保存も可能です。
④karte_v5.html の使い方
🌟 大幅改善：学籍番号1つでいつでもカルテが開けます
merge前でも学生マスターから基本情報を表示。merge後は確定日時も自動セット。CSVの読み込みは不要です。
☁️ Cloudから読み込む（推奨・最速）
🔑
ログイン → 「面談準備カルテ」モーダルが表示される
ログイン後すぐにモーダルが開きます。
☁️
「③ Cloudから読み込む ⭐推奨」を選択
学籍番号を入力してEnterキーまたはボタンを押します。
🔍
2段階で自動検索される
① karte/（面談カルテ）を検索
　→ あれば：面談カルテあり バッジ付きで確定日時＋過去カルテを表示

② なければ students/（学生マスター）を検索
　→ あれば：学生マスターから バッジ付きで氏名・ゼミ・担当職員等を表示
　　「面談日程は未確定です。カルテの土台を作成します。」と案内

プレビューを確認して「カルテを開く」ボタンで完了。
✅ 次の学生への切り替えは？
ヘッダーの「☁️ Cloud読込」ボタンを押して学籍番号を入力するだけ。CSVを再読み込みする必要はありません。
💾 カルテの保存
💾
「💾 保存」ボタン（ヘッダー右上）
ローカルJSONとFirebase Cloudの両方に同時保存。面談中は随時押してください。3分ごとにローカル自動バックアップもされます。
🎙️
音声入力（Chrome推奨）
各テキストエリアの「🎙️」ボタンで音声入力開始。面談記録・カルテメモの長文入力に便利です。

「Firebase接続中…」から変わらない ネットワーク不安定 ページを再読み込み（F5）
index.htmlの日付が全部赤くなる admin.htmlの保存忘れ admin.htmlで再度「保存」を押す
student_master_importで「Firebase未接続」と表示される ローカルファイルで開いている GitHub PagesのURL（https://）でアクセスしてください
